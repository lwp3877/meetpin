## 🎈 프로젝트란?

**밋핀(MeetPin)**은 지도에서 가까운 사람들과 만날 수 있는 앱입니다.

**주요 기능:**
- 🗺️ 지도에서 만남 장소 표시
- 💬 실시간 채팅
- 👤 프로필 관리
- 🔔 알림 받기

## 📌 중요한 파일들

### 1. package.json - 도구 목록
프로젝트에 필요한 모든 도구와 명령어가 적혀있어요.

```json
{
  "scripts": {
    "dev": "개발 서버 시작",
    "build": "배포용 빌드",
    "test": "테스트 실행"
  }
}
```

### 2. CLAUDE.md - AI 도우미 설명서
AI가 프로젝트를 이해하고 도와줄 수 있도록 모든 정보가 담겨있어요.

### 3. .env.local - 비밀 정보
데이터베이스 주소, 비밀번호 등이 들어있어요. (절대 공개하면 안돼요!)

### 4. src/types/global.d.ts - 타입 정의
모든 데이터의 모양을 정의해요.

```typescript
// 사용자 타입
interface User {
  uid: string          // 고유번호
  email: string        // 이메일
  nickname: string     // 닉네임
}

// 모임 타입
interface Room {
  id: string           // 고유번호
  title: string        // 제목
  lat: number          // 위도
  lng: number          // 경도
  max_participants: number  // 최대 인원
}
```

## 🚀 개발 시작하기

### 1단계: 도구 설치

```bash
# Node.js가 설치되어 있어야 해요
# pnpm 설치
npm install -g pnpm

# 프로젝트 도구 설치
pnpm install
```

### 2단계: 비밀 정보 설정

`.env.local` 파일을 만들고 필요한 정보를 입력해요:

```env
# Supabase (데이터베이스)
NEXT_PUBLIC_SUPABASE_URL=your_url
NEXT_PUBLIC_SUPABASE_ANON_KEY=your_key

# Kakao Maps (지도)
NEXT_PUBLIC_KAKAO_JAVASCRIPT_KEY=your_key

# Stripe (결제)
STRIPE_SECRET_KEY=your_key
```

### 3단계: 개발 서버 시작

```bash
pnpm dev
```

이제 브라우저에서 `http://localhost:3001`로 접속하면 돼요!

## 🎮 자주 사용하는 명령어

### 개발 관련

```bash
pnpm dev              # 개발 서버 시작 (가장 많이 씀!)
pnpm build            # 배포용 빌드
pnpm start            # 배포용 서버 시작
```

### 코드 품질 검사

```bash
pnpm typecheck        # 타입 오류 확인
pnpm lint             # 코드 스타일 확인
pnpm lint:fix         # 코드 스타일 자동 수정
pnpm format           # 코드 정리
```

### 테스트

```bash
pnpm test             # 단위 테스트
pnpm e2e              # E2E 테스트
pnpm repo:doctor      # 모든 검사 한번에!
```

## 🔧 문제 해결

### 문제 1: 포트 3001이 이미 사용 중

```bash
# Windows
npx kill-port 3001

# Mac/Linux
lsof -ti:3001 | xargs kill
```

### 문제 2: 빌드 오류

```bash
# 캐시 삭제 후 재설치
rm -rf .next node_modules
pnpm install
pnpm build
```

### 문제 3: 타입 오류

```bash
# TypeScript 검사
pnpm typecheck

# 자동 수정 시도
pnpm lint:fix
```

### 문제 4: 테스트 실패

```bash
# 테스트만 실행
pnpm test

# 특정 파일만 테스트
pnpm test 파일명
```

```
✅ 좋은 예:
- components/RoomCard.tsx      (컴포넌트)
- lib/date-utils.ts            (유틸리티)
- app/map/page.tsx              (페이지)

❌ 나쁜 예:
- components/ROOMCARD.tsx       (대문자만)
- lib/DateUtils.ts              (카멜케이스)
```

```typescript
// ✅ 좋은 예

// ❌ 나쁜 예
const UserName = "홍길동"           // X

```
